# Interpolation

Interpolation embeds JavaScript expression output alongside raw template text.
Edge uses double curly braces as delimiters.

```edge
Hello {{ username }}!
```

Any valid JavaScript expression can be evaluated inside interpolation.

```edge
{{ user.username }}
{{ user.username.toUpperCase() }}
{{ (2 + 2) * 3 }}
{{ (await getUser()).username }}
```

## Multiline Expressions

Expressions may span multiple lines, but the opening and closing double curly braces must each stay on the same line.

```edge
{{
  users.map((user) => {
    return user.username
  })
}}
```

Do not split either delimiter across lines.

```edge
{{
  users.map((user) => user.username)
}
}
```

## Stringified Output

Template output is always a string, so Edge converts expression output with `String()`.
Arrays therefore render as comma-separated values by default.

```edge
{{
  users.map((user) => {
    return user.username
  })
}}
```

Output:

```txt
virk,romain,julien,michael
```

When formatting matters, convert explicitly.

```edge
Hello {{
  users.map((user) => {
    return user.username
  }).join(', ')
}}
```

## Escaped HTML Output

Expression output is HTML-escaped by default to reduce XSS risk.

```edge
{{
  '<span style="color: red">This should be red.</span>'
}}
```

Output:

```txt
&lt;span style="color: red"&gt;This should be red.&lt;/span&gt;
```

Use unescaped output only for trusted HTML that has already been sanitized or generated by trusted code.
Either use triple curly braces or `html.safe()`.

```edge
{{{
  '<span style="color: red">This should be red.</span>'
}}}
```

```edge
{{
  html.safe(
    '<span style="color: red">This should be red.</span>'
  )
}}
```

## Skipping Curly Braces

When Edge is used with frontend code that also uses double curly braces, prefix the braces with `@` to keep Edge from evaluating them.

```edge
Edge should not parse @{{ username }}
```

Output:

```edge
Edge should not parse {{ username }}
```
